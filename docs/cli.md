# CLI Reference

The SuperApp CLI provides project scaffolding, page generation, router code generation, and dev/build commands.

```
superapp -- Elm-inspired TypeScript framework

Commands:
  new <name>         Create a new project
  add <pattern>      Add a page (e.g., "users/[id]")
  gen                Generate router from src/pages/
  dev                Start dev server
  build              Build for production

Options:
  --jsx              Generate .tsx files (with add/new)
  --help             Show help
```

---

## superapp new

Create a new project with a complete starter template.

```bash
superapp new my-app
superapp new my-app --jsx
```

### What it generates

```
my-app/
  index.html
  package.json
  tsconfig.json
  vite.config.ts
  src/
    app.css                  # Tailwind CSS v4 import
    main.ts (or .tsx)        # App entry with routerApp()
    shared.ts                # Shared state type + initial value
    pages/
      Home.ts (or .tsx)      # Home page (/)
      About.ts (or .tsx)     # About page (/about)
      NotFound.ts (or .tsx)  # 404 handler
    generated/
      router.ts              # Auto-generated router (from gen)
```

### Stack

- **Runtime**: SuperApp (TEA + typed router)
- **Bundler**: Vite
- **CSS**: Tailwind CSS v4
- **Type checking**: TypeScript (strict mode)
- **JSX**: Automatic transform via `superapp/jsx-runtime` (configured in tsconfig)

### After creation

```bash
cd my-app
bun install
superapp dev
```

### Options

| Flag | Description |
|------|-------------|
| `--jsx` | Generate `.tsx` files instead of `.ts` |

## superapp add

Add a new page file and regenerate the router.

```bash
superapp add <pattern> [--jsx]
```

### Pattern syntax

The pattern describes both the file path and the route.

| Pattern | File created | Route |
|---------|-------------|-------|
| `About` | `src/pages/About.ts` | `/about` |
| `Home` | `src/pages/Home.ts` | `/` |
| `NotFound` | `src/pages/NotFound.ts` | 404 handler |
| `users/[id]` | `src/pages/users/[id].ts` | `/users/:id` |
| `users/[id:int]` | `src/pages/users/[id:int].ts` | `/users/:id` (int) |
| `users/[id:float]` | `src/pages/users/[id:float].ts` | `/users/:id` (float) |
| `users/[id]/Edit` | `src/pages/users/[id]/Edit.ts` | `/users/:id/edit` |
| `Blog/Index` | `src/pages/Blog/Index.ts` | `/blog` |
| `UserProfile` | `src/pages/UserProfile.ts` | `/user-profile` |

### Dynamic segments

- `[name]` -- String parameter (default)
- `[name:int]` -- Integer parameter
- `[name:float]` -- Float parameter

### Name conventions

- CamelCase filenames are converted to kebab-case routes
- `Home` is special: always maps to `/`
- `NotFound` is special: used as the 404 handler
- `Index` maps to the parent directory route

### Generated file

The command creates a page scaffold:

```ts
import { h, type Dispatch } from "superapp";
import { type PageConfig } from "superapp/router";
import type { Shared } from "../shared";

export const page: PageConfig<{}, never, Shared, {}> = {
  init: () => ({}),
  update: (model) => model,
  view: (model, shared) =>
    h("div", { class: "prose max-w-none" },
      h("h1", {}, "/about"),
    ),
};
```

For pages with parameters, the scaffold includes typed params:

```ts
export const page: PageConfig<{ id: number }, never, Shared, { id: number }> = {
  init: (params) => ({ ...params }),
  update: (model) => model,
  view: (model, shared) =>
    h("div", { class: "prose max-w-none" },
      h("h1", {}, "/users/:id"),
    ),
};
```

### Auto-gen

After creating the page file, `superapp add` automatically runs `superapp gen` to update the router.

| Flag | Description |
|------|-------------|
| `--jsx` | Generate `.tsx` file instead of `.ts` |

## superapp gen

Scan `src/pages/` and generate `src/generated/router.ts`.

```bash
superapp gen
```

### What it does

1. Recursively scans `src/pages/` for `.ts` and `.tsx` files
2. Parses file names into route patterns and parameter types
3. Sorts routes (static before dynamic, more-specific first)
4. Generates a router module that imports all pages and creates a `createRouter` call
5. Writes to `src/generated/router.ts`

### When to run

- After manually creating or renaming page files
- The `add`, `dev`, and `build` commands run `gen` automatically

### Route sorting rules

1. Static routes before dynamic routes (routes with parameters)
2. More segments before fewer segments (more specific first)
3. Alphabetical as tiebreaker

This ensures `/users/settings` is matched before `/users/:id`.

### Output format

```ts
// AUTO-GENERATED by superapp gen -- do not edit
import { createRouter, route, page, str, int } from "superapp/router";
import type { Shared } from "../shared";
import { initialShared } from "../shared";

import { page as Home } from "../pages/Home";
import { page as UsersId } from "../pages/users/[id]";
import { page as NotFound } from "../pages/NotFound";

export const router = createRouter<Shared>({
  routes: [
    page(route("/"), Home),
    page(route("/users/:id", { id: str }), UsersId),
  ],
  shared: initialShared,
  notFound: NotFound,
});
```

Only parsers that are actually used (`str`, `int`, `float`) are imported.

## superapp dev

Start the Vite development server with hot module replacement.

```bash
superapp dev
```

1. Runs `superapp gen` to ensure the router is up to date
2. Starts `npx vite` with inherited stdio

The dev server provides:
- Hot module replacement
- TypeScript compilation
- Tailwind CSS processing

---

## superapp build

Build for production.

```bash
superapp build
```

1. Runs `superapp gen` to ensure the router is up to date
2. Runs `npx vite build` for an optimized production bundle

## Project Structure

A typical SuperApp project:

```
my-app/
  index.html                 # HTML entry point
  package.json
  tsconfig.json
  vite.config.ts
  src/
    main.ts                  # App entry (routerApp + layout)
    shared.ts                # Shared state type and initial value
    app.css                  # Tailwind CSS entry
    pages/                   # Page files (scanned by gen)
      Home.ts
      About.ts
      NotFound.ts
      users/
        Index.ts
        [id].ts
        [id:int]/
          Edit.ts
    generated/               # Auto-generated (do not edit)
      router.ts
```

### Conventions

- Every page file must export `page` as a `PageConfig`
- `src/shared.ts` must export `Shared` type and `initialShared` value
- `src/generated/` is auto-generated; do not edit files in this directory
- Add `src/generated/` to `.gitignore` or commit it (either approach works)
